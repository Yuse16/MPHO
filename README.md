# MPHO

## Regalos que conectan personas, historias y ocasiones

MPHO es una plataforma mexicana para descubrir, personalizar, comprar, preparar y entregar regalos mediante una experiencia única coordinada de principio a fin.

Para el cliente, MPHO es la marca principal durante toda la experiencia:

```text
descubrir
→ elegir
→ personalizar
→ pagar
→ preparar
→ entregar
→ dar seguimiento
→ recordar la próxima ocasión
```

El cliente compra en MPHO, recibe comunicaciones de MPHO, sigue su pedido en MPHO y solicita soporte a MPHO.

Los comercios aliados trabajan internamente como **Puntos MPHO**: centros de abastecimiento, inventario, recepción, inspección, preparación, personalización, almacenamiento y distribución. Su identidad no se muestra normalmente al cliente, salvo cuando exista una obligación legal, fiscal, de garantía, seguridad, retiro autorizado o una colaboración comercial aprobada.

---

## Significado de marca

Interpretación recomendada:

> **MPHO — México, Personas, Historias y Ocasiones.**

Porque detrás de cada regalo existe una persona, una historia y una ocasión que merece recordarse.

---

## El problema

Enviar un regalo suele obligar al cliente a coordinar por separado:

- Qué comprar.
- Dónde encontrarlo.
- Si hay existencia.
- Cómo personalizarlo.
- Cómo envolverlo.
- Cómo pagarlo.
- Cómo enviarlo.
- Quién lo recibe.
- Cómo dar seguimiento.
- Qué hacer si algo falla.

Al mismo tiempo, muchos negocios locales tienen productos y experiencia, pero no cuentan con una plataforma completa para:

- Ecommerce.
- Automatización.
- Cobro digital.
- Catálogo.
- Inventario.
- Preparación.
- Personalización.
- Evidencia.
- Entregas.
- Seguimiento.
- Ganancias.
- Soporte.
- Seguridad.
- Analítica.

MPHO conecta todas estas partes en una sola operación.

---

## Propuesta de valor

### Para clientes

- Encontrar mejores regalos en menos tiempo.
- Recibir recomendaciones con HADIA.
- Ver opciones disponibles para su ciudad y fecha.
- Personalizar productos.
- Comprar en una sola plataforma.
- Recibir seguimiento.
- Acceder a entrega programada o rápida.
- Guardar personas, fechas y ocasiones.
- Tener soporte centralizado.

### Para aliados

- Recibir pedidos sin crear su propio ecommerce.
- Aumentar utilización de inventario y capacidad.
- Ganar por vender, recibir, inspeccionar, preparar, personalizar o entregar.
- Operar mediante MPHO Aliados.
- Consultar tareas, tiempos, incidencias y ganancias.
- Crecer sin depender de captar directamente al cliente final.

### Para MPHO

- Crear una red local escalable de preparación y distribución.
- Controlar la experiencia completa.
- Construir una marca propia.
- Unificar datos, operación, pagos, calidad y servicio.
- Expandirse posteriormente a otras ciudades.

---

## Cómo funciona un pedido

```text
Cliente descubre o solicita una recomendación
→ MPHO valida zona, fecha, precio y disponibilidad
→ Cliente personaliza y paga
→ MPHO asigna un Punto MPHO
→ El aliado acepta y confirma capacidad
→ Se reserva producto local o se recibe un producto externo
→ El aliado inspecciona, prepara y documenta
→ MPHO coordina la entrega
→ El cliente recibe seguimiento
→ El destinatario recibe el regalo
→ MPHO concilia costos, ganancias y cierre
```

Cada pedido debe tener:

- Un precio confirmado.
- Un responsable operativo.
- Historial de estados.
- Evidencia cuando corresponda.
- Control de pagos.
- Control de entrega.
- Manejo de incidencias.
- Registro financiero auditable.

## HADIA

HADIA es el asistente inteligente de regalos de MPHO.

Puede ayudar según:

- Persona.
- Relación.
- Ocasión.
- Emoción deseada.
- Presupuesto.
- Fecha.
- Zona.
- Preferencias.
- Personalización.
- Urgencia.

HADIA puede:

- Buscar catálogo autorizado.
- Comparar opciones.
- Explicar recomendaciones.
- Consultar disponibilidad permitida.
- Consultar cobertura.
- Consultar MPHORA.
- Crear propuestas.
- Crear borradores de carrito.
- Escalar a soporte humano.

HADIA no puede:

- Confirmar pagos.
- Aprobar reembolsos.
- Modificar precios.
- Ejecutar payouts.
- Aprobar aliados.
- Comprar productos externos autónomamente.
- Cambiar estados privilegiados.
- Saltarse permisos.
- Revelar datos privados o internos.

---

## MPHORA

MPHORA es la modalidad de entrega rápida de MPHO.

Solo puede mostrarse cuando existen datos suficientemente confiables sobre:

- Producto.
- Variante.
- Existencia.
- Punto MPHO disponible.
- Capacidad de preparación.
- Zona.
- Horario.
- Personalización.
- Repartidor.
- Vigencia de la cotización.

MPHORA nunca debe prometer una entrega rápida basándose únicamente en una imagen, un catálogo desactualizado o una suposición.

## Puntos MPHO

Un Punto MPHO puede tener capacidades diferentes:

- Suministro de producto.
- Inventario.
- Recepción de paquetes.
- Inspección.
- Almacenamiento temporal.
- Envoltura.
- Preparación.
- Personalización.
- Impresión.
- Entrega al repartidor.
- Entrega propia autorizada.

Las capacidades se aprueban por separado mediante:

```text
solicitud
→ verificación
→ acuerdo
→ capacitación
→ pedido de prueba
→ activación
→ monitoreo
```

## Tipos de producto

### Producto local

Producto suministrado por un Punto MPHO.

Ejemplos:

- Flores.
- Chocolates.
- Globos.
- Tazas.
- Peluches.
- Cajas.
- Detalles personalizados.

### Producto externo o por encargo

Producto adquirido a través de un proveedor externo autorizado.

```text
Cliente compra en MPHO
→ MPHO coordina la compra autorizada
→ Proveedor envía a un Punto MPHO
→ Punto MPHO recibe e inspecciona
→ Punto MPHO prepara
→ MPHO coordina entrega
```

En el MVP no se permiten compras externas autónomas ejecutadas por IA.

### Paquete combinado

Conjunto prediseñado de productos y servicios.

En el MVP, un solo Punto MPHO debe ser responsable operativo principal por pedido.

## Fuera del MVP

No construir inicialmente:

- Operación nacional.
- Marketplace abierto sin revisión.
- División compleja de un pedido entre varios aliados.
- Scraping indiscriminado de marketplaces.

- Franquicias.

## Seguridad

Principios obligatorios:

- Denegación por defecto.
- RLS para datos privados.
- Aislamiento entre clientes.
- Aislamiento entre aliados.
- Permisos internos por rol.
- MFA para cuentas privilegiadas.
- Validación en servidor.
- Webhooks verificados.
- Idempotencia.
- Archivos privados.
- URLs firmadas de corta duración.
- Auditoría.
- Historial financiero.
- No guardar secretos en Git.
- No usar claves elevadas en el navegador.
- No cachear datos privados o financieros.
- Reautenticación para acciones sensibles.
- Separación de funciones.
- Respaldo y recuperación.
- Respuesta a incidentes.

La documentación de seguridad reduce riesgos, pero no garantiza que nunca pueda ocurrir un ataque, fraude o incidente.

## Regla principal

MPHO debe resolver la experiencia completa de regalar sin obligar al cliente a coordinar tiendas, proveedores, envoltura, personalización y repartidores por separado.

La tecnología automatiza lo repetitivo.

El control humano permanece en:

- Compras externas.
- Calidad.
- Reembolsos.
- Payouts.
- Incidentes.
- Excepciones.
- Seguridad.
- Decisiones sensibles.

> **El cliente recuerda a MPHO.  
> Los aliados crecen dentro de la red MPHO.  
> La plataforma controla la experiencia completa.**
